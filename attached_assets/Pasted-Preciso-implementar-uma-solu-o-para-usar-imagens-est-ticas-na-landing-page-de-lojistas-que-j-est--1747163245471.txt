Preciso implementar uma solução para usar imagens estáticas na landing page de lojistas que já está criada, sem modificar o sistema atual de upload e exibição de imagens.

Por favor, execute as seguintes tarefas:

## 1. Criar estrutura de pastas para imagens estáticas

Crie as seguintes pastas no diretório public:
- /public/landing/
- /public/landing/sellers/

## 2. Adicionar imagens de exemplo para a landing page de lojistas

Baixe e adicione 3-4 imagens relacionadas a comércio, lojas e vendedores para a pasta /public/landing/sellers/:
- Imagens relacionadas a lojistas, comércio e vendas
- Renomeie com nomes descritivos como "hero-banner.jpg", "feature-commission.jpg", "feature-dashboard.jpg"
- Otimize as imagens para web (tamanho máximo de 500KB cada)

## 3. Criar um componente simples para imagens das landing pages

Crie um novo componente em client/src/components/landing/LandingImage.tsx:

```tsx
interface LandingImageProps {
  src: string;
  alt?: string;
  className?: string;
  width?: number;
  height?: number;
}

export function LandingImage({ 
  src, 
  alt = "", 
  className = "", 
  width, 
  height 
}: LandingImageProps) {
  // Não usa a lógica de SafeImage, simplesmente renderiza a imagem diretamente
  return (
    <img 
      src={src} 
      alt={alt}
      className={`rounded-lg ${className}`}
      loading="lazy"
      decoding="async"
      width={width}
      height={height}
    />
  );
}
4. Modificar a página de landing de lojistas existente
Localize o arquivo da landing page de lojistas (provavelmente em pages/landing/sellers.tsx ou similar).
Substitua qualquer componente SafeImage ou referências a URLs de imagem pelo novo componente LandingImage, apontando para as imagens estáticas:
Exemplo de substituição:
tsx// Substitua isso:
<SafeImage 
  src={sellerImage} 
  alt="Lojista gerenciando produtos"
  className="w-full"
/>

// Por isso:
<LandingImage 
  src="/landing/sellers/hero-banner.jpg" 
  alt="Lojista gerenciando produtos"
  className="w-full"
/>
Faça isso para todas as imagens na página, usando os caminhos para as novas imagens estáticas que você adicionou.
5. Teste as alterações
Verifique se a landing page está carregando corretamente com as novas imagens estáticas, sem afetar o sistema atual de imagens em outras partes do aplicativo.
IMPORTANTE: Não modifique nenhum componente existente de imagem (como SafeImage) ou a lógica atual de upload e armazenamento. As imagens para as landing pages devem ser completamente independentes do sistema atual.
